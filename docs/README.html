3.2. Steps to test AAP 2 on Ansible Advanced - Lab CI (Test Env)

  1. You can install ansible-core and and ansible-navigator on bastion as follows.

    $ sudo yum install ansible-core ansible-navigator -y

  2. Test pinging all of the hosts using ansible-navigator.

    $ cat > $HOME/ping.yml <<EOF
---
- hosts: all
  gather_facts: false
  tasks:
    - ping:
EOF

  3. You need to perform image registry authenication before you run ansible-navigator.

    $ podman login -u your_email_id@redhat.com registry.redhat.io
password: <enter your password>

  4. Execute ansible-navigate as follows.

    $ ansible-navigator run  ping.yml -i /etc/ansible/hosts -u cloud-user -e ansible_ssh_private_key_file=.ssh/${GUID}key.pem

3.3. Test AAP 2 with Multitier deployment.

  1. Clone Multitier Application from following git repository on bastion.

    $ cd $HOME
$ git clone https://github.com/tonykay/multi-tier-app-deployer.git
$ cd multi-tier-app-deployer

  2. Copy inventory file in repository directory as follows.

    $ cp /etc/ansible/hosts .

  3. Replace ansible.cfg file in cloned repository with following content as follows.

    $ cat > ansible.cfg <<EOF
[defaults]
inventory               = hosts
remote_user             = cloud-user
private_key_file        = .ssh/${GUID}key.pem

EOF

  4. Now copy $HOME/.ssh directory in the git cloned repository as follows.

    $ cp -r $HOME/.ssh .

  5. Install collection in current directory as follows.

    $ ansible-galaxy collection install -r requirements.yml -p collections

  6. Perform image registry authenication before you can run ansible-navigator.

    $ podman login -u your_email_id@redhat.com registry.redhat.io
password: <enter your password>

  7. You can now execute multtier deploy-app.yml playbook with ansible-navigator as follows.

    $ ansible-navigator -m stdout run deploy-app.yml

  8. You can also test teardown-app.yml as follows.

    $ ansible-navigator -m stdout run teardown-app.yml
